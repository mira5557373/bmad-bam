## Common Alerts

### Alert: High Error Rate

**Trigger:** Error rate > 5% for 5 minutes

**Investigation:**
1. Check application logs: `kubectl logs -l app={{service_name}} --tail=100`
2. Check recent deployments: `kubectl rollout history deployment/{{service_name}}`
3. Check dependency health: Deep health endpoint
4. Check database connections: `./scripts/check-db-connections.sh`

**Resolution:**
- If deployment related → Rollback
- If database related → Check connection pool, failover
- If external dependency → Enable fallback/circuit breaker

---

### Alert: High Latency

**Trigger:** p99 latency > 2x baseline for 5 minutes

**Investigation:**
1. Check slow queries: `./scripts/slow-query-log.sh`
2. Check cache hit rate: Grafana dashboard
3. Check resource utilization: `kubectl top pods -l app={{service_name}}`
4. Check external API latency: APM traces

**Resolution:**
- If database related → Add indexes, optimize queries
- If cache related → Investigate evictions, warm cache
- If resource related → Scale horizontally

---

### Alert: Tenant Isolation Violation

**Trigger:** Cross-tenant access attempt detected

**Investigation:**
1. Identify affected request: Audit logs
2. Check tenant context propagation: Trace logs
3. Verify RLS policies: `./scripts/verify-rls.sh`

**Resolution:**
- Immediate: Block affected endpoint
- Root cause: Fix tenant context handling
- Verification: Run isolation tests

## Operational Procedures

### Procedure: Scale Up

**When:** High traffic, approaching capacity

```bash
# Check current replicas
kubectl get deployment {{service_name}} -n production

# Scale up
kubectl scale deployment {{service_name}} --replicas=X -n production

# Verify
kubectl get pods -l app={{service_name}} -n production -w
```

### Procedure: Force Pod Restart

**When:** Pod in bad state, memory leak

```bash
# Restart single pod
kubectl delete pod {{pod_name}} -n production

# Rolling restart all pods
kubectl rollout restart deployment/{{service_name}} -n production
```

### Procedure: Database Failover

**When:** Primary database unavailable

```bash
# Promote read replica (AWS RDS example)
aws rds promote-read-replica --db-instance-identifier {{replica_id}}

# Update connection string
kubectl set env deployment/{{service_name}} DATABASE_URL={{new_url}}
```
