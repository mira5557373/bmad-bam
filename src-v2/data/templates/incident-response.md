## Incident Workflow

### 1. Detection & Triage

- [ ] Alert received and acknowledged
- [ ] Initial severity assigned
- [ ] Incident channel created: `#incident-{date}-{id}`
- [ ] Tenant impact assessment started

### 2. Investigation

- [ ] Affected tenants identified
- [ ] Root cause investigation initiated
- [ ] Timeline of events documented
- [ ] Relevant logs collected

### 3. Mitigation

- [ ] Mitigation options evaluated
- [ ] Mitigation applied
- [ ] Tenant communication sent (if P1/P2)
- [ ] Monitoring for resolution

### 4. Resolution

- [ ] Service restored
- [ ] All affected tenants verified operational
- [ ] Incident status updated
- [ ] Post-incident review scheduled

## Tenant Communication Templates

Next Update: Within [X] minutes
