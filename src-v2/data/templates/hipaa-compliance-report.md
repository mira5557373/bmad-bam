## Multi-Tenant PHI Isolation

<!-- FILL: Document tenant isolation for PHI -->
| Layer | Control | Verification |
|-------|---------|--------------|
| Application | TenantContext | Unit tests |
| Database | RLS policies | Integration tests |
| Storage | Tenant-prefixed paths | Access tests |
| Logs | Tenant-scoped | Audit review |
| Backups | Tenant-isolated | Restore tests |

### Tenant Isolation Configuration

```yaml
hipaa_isolation:
  database:
    model: rls  # or schema_per_tenant for high isolation
    encryption: tenant_specific_keys
    
  storage:
    bucket_per_tenant: true
    encryption: customer_managed_keys
    
  logging:
    phi_redaction: enabled
    tenant_scope: strict
```
