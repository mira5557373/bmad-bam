## Tier Details

### L0 - Immediate Memory

<!-- FILL: Document immediate memory -->
| Property | Value |
|----------|-------|
| Purpose | Current turn context |
| Capacity | Context window limit |
| Access | Automatic |
| Tenant Isolation | Inherent (request scope) |

### L1 - Working Memory

| Property | Value |
|----------|-------|
| Purpose | Agent run state, tool results |
| Capacity | Configurable (default 100 items) |
| Access | Agent-managed |
| Tenant Isolation | Run ID + Tenant ID |

**Structure:**
```json
{
  "run_id": "{{run_id}}",
  "tenant_id": "{{tenant_id}}",
  "items": [
    {
      "key": "tool_result_1",
      "value": {...},
      "timestamp": "{{timestamp}}"
    }
  ]
}
```

### L2 - Session Memory

| Property | Value |
|----------|-------|
| Purpose | Conversation history, preferences |
| Capacity | Configurable per tier |
| Access | Session-scoped |
| Tenant Isolation | Tenant ID prefix in Redis |

**Key Pattern:** `tenant:{{tenant_id}}:session:{{session_id}}:memory`

### L3 - Long-term Memory

| Property | Value |
|----------|-------|
| Purpose | User preferences, learned patterns |
| Capacity | Quota per tier |
| Access | Cross-session |
| Tenant Isolation | RLS on memory table |
