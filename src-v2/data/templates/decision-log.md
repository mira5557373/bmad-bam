## ADR-001: Tenant Isolation Model

**Status:** Accepted
**Date:** {{date}}
**Deciders:** Platform Architecture Team

#### Context

We need to determine how tenant data will be isolated in the database layer. This decision impacts security, scalability, operations, and compliance posture.

#### Decision Drivers

- Security: Prevent cross-tenant data leakage
- Compliance: Meet SOC2 and GDPR requirements
- Operations: Balance complexity vs. tenant count
- Cost: Optimize infrastructure spend
- Performance: Maintain query performance at scale

#### Considered Options

1. **Row-Level Security (RLS):** Shared tables with PostgreSQL RLS policies
2. **Schema-per-Tenant:** Separate schema per tenant in shared database
3. **Database-per-Tenant:** Dedicated database per tenant
