## Upgrade Process

<!-- FILL: Document model upgrade workflow -->
### Pre-Upgrade

| Step | Action | Owner |
|------|--------|-------|
| 1 | Evaluate new model version | AI Team |
| 2 | Run benchmark suite | AI Team |
| 3 | Canary deployment (5% traffic) | Platform |
| 4 | Monitor metrics for 48h | Platform |

### Upgrade Execution

| Phase | Scope | Rollback Window |
|-------|-------|-----------------|
| Canary | 5% traffic | Immediate |
| Early adopters | PRO tier opt-in | 24 hours |
| General | All FREE tier | 7 days |
| Enterprise | Scheduled per tenant | Per contract |

### Rollback Triggers

| Metric | Threshold | Action |
|--------|-----------|--------|
| Error rate | > 5% | Auto-rollback |
| Latency p99 | > 2x baseline | Alert + manual |
| Quality score | < 90% baseline | Alert + manual |

## Model Configuration

```yaml
llm_config:
  default_model: "gpt-4-turbo-2024-04-09"
  fallback_model: "gpt-4o-mini-2024-07-18"
  
  routing:
    high_complexity:
      model: "gpt-4-turbo"
      max_tokens: 4096
    low_complexity:
      model: "gpt-4o-mini"
      max_tokens: 1024
  
  tenant_overrides:
```
