# Step 03: Map AI Capability Consumers

## Purpose

Identify and profile all stakeholders who will directly consume, interact with, or benefit from AI platform capabilities. This mapping ensures the platform design addresses actual user needs and that adoption strategies target the right audiences within each tenant organization.

## Prerequisites

- [ ] Step 01 (Identify Tenant Stakeholders) is complete
- [ ] Step 02 (Analyze Decision Makers) is complete
- [ ] AI platform capability catalog or roadmap is available
- [ ] Understanding of intended AI use cases per tenant type
- [ ] Access to user research or persona documentation (if available)

## Actions

1. **Catalog AI Platform Capabilities**
   - List current and planned AI capabilities:
     - Natural Language Processing (chatbots, text analysis, summarization)
     - Machine Learning Operations (model training, deployment, monitoring)
     - Predictive Analytics (forecasting, recommendations)
     - Computer Vision (image analysis, document processing)
     - Agentic AI (autonomous task execution, workflow automation)
   - Group capabilities by consumption pattern (API, UI, embedded, automated)

2. **Identify Direct AI Consumers by Tenant Type**
   - **Enterprise Consumers:**
     - Data Scientists: Model development, training, experimentation
     - ML Engineers: Model deployment, pipeline management
     - Business Analysts: Insights consumption, report generation
     - Application Developers: API integration, feature embedding
     - End Users: AI-powered application features
   - **SMB Consumers:**
     - Power Users: Pre-built AI tools and templates
     - Business Owners: Automated insights and recommendations
     - Customer-facing Staff: AI-assisted customer interactions
   - **Government Consumers:**
     - Analysts: Pattern detection, anomaly identification
     - Case Workers: AI-assisted decision support
     - Public Services: Automated citizen interactions

3. **Map Consumption Patterns**
   - Document how each consumer group will interact with AI:
     - Direct API consumers (developers, data scientists)
     - UI/Dashboard consumers (analysts, business users)
     - Embedded AI consumers (end users via applications)
     - Automated/Background consumers (systems, workflows)
   - Identify consumption frequency and volume expectations

4. **Analyze Consumer Requirements**
   - For each consumer group, document:
     - Technical skill level and AI literacy
     - Primary use cases and jobs-to-be-done
     - Performance expectations (latency, accuracy, reliability)
     - Customization and configuration needs
     - Support and training requirements

5. **Identify Consumer Pain Points**
   - Document current challenges without AI capabilities
   - Identify friction points with existing AI tools (if any)
   - Note specific concerns about AI adoption (trust, accuracy, job displacement)
   - Map resistance factors by consumer segment

6. **Create Consumer Personas**
   - Develop detailed personas for primary consumer types:
     - Background and role context
     - AI capability needs and goals
     - Technical proficiency level
     - Adoption barriers and motivations
     - Success metrics from their perspective

7. **Map Consumer to Capability Matrix**
   - Create matrix showing which consumers use which capabilities
   - Identify capability gaps based on consumer needs
   - Highlight high-priority consumer-capability combinations
   - Note dependencies between capabilities and consumers

## Verification

- [ ] All AI platform capabilities have identified consumer groups
- [ ] Consumer personas represent major user segments
- [ ] Consumption patterns are documented for each consumer type
- [ ] Consumer requirements include skill levels and support needs
- [ ] Consumer-Capability Matrix is complete and validated

## Outputs

| Output | Description | Format |
|--------|-------------|--------|
| AI Capability Catalog | Comprehensive list of platform AI capabilities | Document |
| Consumer Personas | Detailed profiles of AI capability consumers | Document |
| Consumer-Capability Matrix | Mapping of consumers to capabilities | Spreadsheet |
| Consumption Pattern Analysis | How different groups interact with AI | Document |
| Consumer Requirements Summary | Aggregated needs by consumer segment | Document |
