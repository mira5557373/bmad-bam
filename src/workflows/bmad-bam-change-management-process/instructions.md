# Change Management Process - Instructions

## Overview

This workflow guides the design of change management processes for multi-tenant AI platforms. It covers change classification, tenant impact assessment, approval workflows, and rollback procedures.

## Quick Start

1. Ensure master architecture is defined
2. Run change management process workflow
3. Follow step-by-step guidance
4. Produce change management documentation

## Workflow Steps

### Create Mode (Steps 01-04)
1. **Classify Changes** - Define change types and risk levels
2. **Tenant Impact** - Design tenant impact assessment
3. **Approval Workflow** - Design approval processes
4. **Assembly** - Compile final change management document

### Edit Mode (Steps 10-11)
1. **Load Existing** - Load current change management process
2. **Apply Changes** - Make targeted modifications

### Validate Mode (Steps 20-22)
1. **Load Artifact** - Load process for validation
2. **Validate** - Run quality checks
3. **Report** - Generate validation report

## Key Deliverables

- Change management process document
- Change approval matrix
- Tenant impact assessment template
- Rollback procedures
