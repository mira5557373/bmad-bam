# Chaos Engineering Design - Instructions

## Overview

This workflow guides the design of chaos engineering practices for multi-tenant AI platforms. It covers experiment design, blast radius control, tenant isolation verification, and recovery testing.

## Quick Start

1. Ensure master architecture is defined
2. Run chaos engineering design workflow
3. Follow step-by-step guidance
4. Produce chaos engineering documentation

## Workflow Steps

### Create Mode (Steps 01-03)
1. **Design Experiments** - Define chaos experiments and hypotheses
2. **Blast Radius** - Design blast radius controls and safety
3. **Assembly** - Compile final chaos engineering document

### Edit Mode (Steps 10-11)
1. **Load Existing** - Load current chaos engineering design
2. **Apply Changes** - Make targeted modifications

### Validate Mode (Steps 20-22)
1. **Load Artifact** - Load design for validation
2. **Validate** - Run quality checks
3. **Report** - Generate validation report

## Key Deliverables

- Chaos engineering design document
- Experiment catalog
- Blast radius controls
- Safety procedures
