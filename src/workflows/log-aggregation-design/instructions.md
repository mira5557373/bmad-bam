# Log Aggregation Design - Instructions

## Overview

This workflow guides the design of centralized log aggregation for multi-tenant AI platforms. It covers log collection, tenant isolation, retention policies, and search/analysis.

## Quick Start

1. Ensure master architecture is defined
2. Run log aggregation design workflow
3. Follow step-by-step guidance
4. Produce log aggregation documentation

## Workflow Steps

### Create Mode (Steps 01-03)
1. **Collection Design** - Design log collection and pipeline
2. **Tenant Isolation** - Design tenant-isolated access
3. **Assembly** - Compile final log aggregation document

### Edit Mode (Steps 10-11)
1. **Load Existing** - Load current log aggregation design
2. **Apply Changes** - Make targeted modifications

### Validate Mode (Steps 20-22)
1. **Load Artifact** - Load design for validation
2. **Validate** - Run quality checks
3. **Report** - Generate validation report

## Key Deliverables

- Log aggregation design document
- Log retention policy
- Access control design
- Search/analysis specifications
