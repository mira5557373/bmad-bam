# Step 2: Apply Targeted Modifications

Apply requested changes to the contract evolution plan.

## Tasks

1. **Identify Affected Components**
   - Determine which evolution documents are impacted
   - Check if changes affect the version strategy
   - Note if breaking change assessment needs update

2. **Present Current State**
   - Show relevant sections of the evolution plan
   - Highlight specific items to be changed

3. **Apply Modifications**
   - Update contract changes as requested
   - Modify migration guide if needed
   - Adjust deprecation timeline if applicable
   - Update version strategy ADR if significant

4. **Reassess Breaking Changes**
   If changes were modified:
   - Re-run breaking change detection
   - Update impact analysis
   - Verify migration path still valid

5. **Verify Evolution Consistency**
   - Ensure version bump still appropriate
   - Confirm deprecation timeline is realistic
   - Check migration guide reflects current changes

## Output

Write updated evolution documents to their original locations.

Present a diff summary showing:
- Modified changes in the manifest
- Updated breaking change assessment
- Timeline adjustments
- Version strategy changes (if any)

Ask for confirmation before finalizing.
