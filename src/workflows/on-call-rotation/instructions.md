# On-Call Rotation - Instructions

## Overview

This workflow guides the design of on-call rotation and escalation procedures for multi-tenant AI platforms. It covers rotation scheduling, escalation policies, tenant-priority handling, and burnout prevention.

## Quick Start

1. Ensure master architecture is defined
2. Run on-call rotation workflow
3. Follow step-by-step guidance
4. Produce on-call rotation documentation

## Workflow Steps

### Create Mode (Steps 01-03)
1. **Rotation Design** - Design rotation schedules and coverage
2. **Escalation Policy** - Design escalation and tenant routing
3. **Assembly** - Compile final on-call rotation document

### Edit Mode (Steps 10-11)
1. **Load Existing** - Load current on-call rotation design
2. **Apply Changes** - Make targeted modifications

### Validate Mode (Steps 20-22)
1. **Load Artifact** - Load design for validation
2. **Validate** - Run quality checks
3. **Report** - Generate validation report

## Key Deliverables

- On-call rotation design document
- Escalation policy
- Rotation calendar
- Burnout prevention guidelines
