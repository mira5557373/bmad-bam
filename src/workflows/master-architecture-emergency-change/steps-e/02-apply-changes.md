# Step 2: Apply Targeted Modifications

Based on the user's requested changes:

1. Identify the affected phase in the emergency change process
2. Present the current content of each affected document
3. Apply the requested modifications while preserving:
   - Emergency ID and tracking
   - Approval chain integrity
   - Audit trail
4. If modifying emergency scope:
   - Re-assess impact
   - May require re-approval
   - Update all dependent documents
5. If modifying implementation plan:
   - Verify approval still valid
   - Update rollback procedures if needed
   - Notify affected teams
6. If adding new information post-implementation:
   - Update implementation log
   - Add to lessons learned
   - Create additional tech debt items if needed
7. Validate the modified documents against quality gates
8. Write updated documents back to their original locations

**CRITICAL:** Changes to approval status require re-running approval process.

Present a diff summary of changes made and ask for confirmation.
