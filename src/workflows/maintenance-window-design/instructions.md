# Maintenance Window Design - Instructions

## Overview

This workflow guides the design of maintenance window scheduling for multi-tenant AI platforms. It covers window planning, tenant coordination, zero-downtime strategies, and communication procedures.

## Quick Start

1. Ensure master architecture is defined
2. Run maintenance window design workflow
3. Follow step-by-step guidance
4. Produce maintenance window documentation

## Workflow Steps

### Create Mode (Steps 01-03)
1. **Window Planning** - Define maintenance windows and strategies
2. **Tenant Coordination** - Design tenant notification and coordination
3. **Assembly** - Compile final maintenance window document

### Edit Mode (Steps 10-11)
1. **Load Existing** - Load current maintenance window design
2. **Apply Changes** - Make targeted modifications

### Validate Mode (Steps 20-22)
1. **Load Artifact** - Load design for validation
2. **Validate** - Run quality checks
3. **Report** - Generate validation report

## Key Deliverables

- Maintenance window design document
- Maintenance schedule
- Communication templates
- Zero-downtime procedures
