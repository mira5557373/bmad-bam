# Storyteller Guide - BAM Extension

**When to load:** During Phase 2-3 (Discovery/Planning) when crafting tenant journey narratives, or when user mentions user stories, tenant lifecycle, or value propositions.
**Integrates with:** Storyteller (bmad-cis-agent-storyteller), narrative design, SaaS storytelling

This guide provides BAM-specific context for storytellers working on multi-tenant agentic AI platforms.

## Role Context

As a storyteller on a BAM project, you focus on:
- Crafting compelling narratives for tenant journeys
- Documenting user flows with tenant context
- Creating stories that span tenant lifecycle
- Communicating platform value per tier

## Core Concepts

### Tenant Journey Narrative
Every tenant experiences a journey from discovery through advocacy. Effective storytelling maps this journey with emotional arcs that resonate at each phase - from the frustration of the initial problem through the satisfaction of success and the pride of becoming an advocate.

### Tier-Differentiated Value Propositions
Each tier requires distinct value narratives that speak to different needs and motivations. Free tier stories emphasize low-risk exploration, Pro tier stories focus on productivity acceleration, and Enterprise stories highlight transformation and strategic partnership.

### Multi-Tenant User Stories
User stories in multi-tenant systems must explicitly include tenant context. The format expands to "As a [role] at a [tier] organization, I want [capability] so that [outcome]" with acceptance criteria that account for tier-specific behaviors.

## Application Guidelines

When crafting tenant narratives:
1. Identify the protagonist persona and their tier context before writing
2. Map the emotional arc from problem to solution to success
3. Include tier-specific variations when experiences differ
4. Connect stories to measurable business outcomes
5. Use real tenant examples (anonymized) to ground narratives in authenticity

## SaaS Narrative Frameworks

Use these frameworks to create compelling stories that resonate with multi-tenant SaaS contexts:

### Tenant Journey Narrative Structure

| Journey Phase | Story Focus | Emotional Arc | Key Messages |
|--------------|-------------|---------------|--------------|
| **Discovery** | Problem recognition | Frustration → Hope | "There's a better way" |
| **Evaluation** | Solution exploration | Curiosity → Interest | "See what's possible" |
| **Onboarding** | Getting started | Excitement → Confidence | "You can do this" |
| **Activation** | First success | Effort → Achievement | "You're getting value" |
| **Expansion** | Growing usage | Satisfaction → Ambition | "Unlock more potential" |
| **Advocacy** | Sharing success | Pride → Community | "Join the movement" |

### Tier-Based Value Narratives

| Tier | Primary Narrative | Value Proposition Focus |
|------|------------------|------------------------|
| **Free** | "Start your journey" | Low risk, quick wins, essential capabilities |
| **Pro** | "Accelerate your success" | Productivity gains, team collaboration, advanced features |
| **Enterprise** | "Transform your organization" | Scale, security, customization, strategic partnership |

## Actionable Guidance

### Creating Tenant Journey Stories

1. **Identify Protagonist** - Define the typical user persona for the target tier
2. **Establish Context** - Describe their current situation and challenges
3. **Introduce Conflict** - Highlight the problem they need to solve
4. **Present Solution** - Show how the platform addresses their needs
5. **Describe Journey** - Walk through key milestones and experiences
6. **Celebrate Success** - Illustrate the positive outcomes achieved
7. **Inspire Next Steps** - Point toward continued growth and engagement

### Documenting Multi-Tenant User Flows

1. **Map Entry Points** - Identify all ways users enter the flow
2. **Note Tenant Context** - Mark where tenant identification occurs
3. **Highlight Tier Variations** - Show where experiences differ by tier
4. **Include Decision Points** - Document user choices and branches
5. **Capture Error Paths** - Don't forget failure scenarios
6. **Show Value Moments** - Highlight where users experience value
7. **Connect to Outcomes** - Link flows to business outcomes

### Writing Effective User Stories for Multi-Tenant

1. **Include Tenant Context** - Specify tier and tenant characteristics
2. **State the Persona** - "As a [role] at a [tier] organization..."
3. **Express the Need** - "I want to [capability]..."
4. **Explain the Value** - "So that I can [outcome]..."
5. **Add Acceptance Criteria** - Include tenant-specific criteria
6. **Note Tier Variations** - Document if story varies by tier
7. **Reference Dependencies** - Link to related stories across modules

## SaaS-Specific Considerations

### Persona Stories by Tier

**Free Tier Personas:**

| Persona | Story Arc | Key Moments |
|---------|-----------|-------------|
| Solo Creator | "Proving the concept" | First successful outcome |
| Small Team Lead | "Testing before committing" | Sharing with team |
| Student/Learner | "Building skills" | Portfolio piece completion |

**Pro Tier Personas:**

| Persona | Story Arc | Key Moments |
|---------|-----------|-------------|
| Team Manager | "Scaling operations" | Team productivity milestone |
| Power User | "Pushing boundaries" | Advanced feature mastery |
| Integration Builder | "Connecting systems" | Workflow automation success |

**Enterprise Tier Personas:**

| Persona | Story Arc | Key Moments |
|---------|-----------|-------------|
| Executive Sponsor | "Driving transformation" | Org-wide adoption |
| IT Administrator | "Ensuring compliance" | Security audit passed |
| Platform Champion | "Building internal adoption" | Center of excellence |

### Upgrade/Downgrade Narratives

**Upgrade Story Arc:**
1. User hitting limits, feeling constrained
2. Discovery of advanced capabilities
3. Evaluation of upgrade benefits vs cost
4. Decision moment with clear value understanding
5. Seamless transition experience
6. Immediate access to new capabilities
7. First "aha" moment with new features

**Downgrade Story Arc (minimize friction):**
1. Acknowledge changing needs
2. Show what will change clearly
3. Offer alternatives (pause vs downgrade)
4. Smooth transition with data preservation
5. Keep door open for return
6. Maintain relationship

### Multi-Tenant Success Story Template

**Structure:**
```
Title: [Compelling headline with outcome]

The Challenge:
- Company context (tier, size, industry)
- Problem they faced
- Previous solutions attempted

The Solution:
- Why they chose this platform
- How they implemented it
- Key features used

The Results:
- Quantified outcomes
- Qualitative improvements
- Quote from stakeholder

The Journey Continues:
- Future plans
- Tier progression
- Advocacy actions
```

### Feature Discovery Narratives

**Progressive Disclosure Story:**
1. Introduce basic capability
2. User achieves initial success
3. Surface related advanced feature
4. Show how it enhances value
5. Guide user to try it
6. Celebrate expanded capability

**Cross-Sell Narrative (Free to Pro):**
| Stage | Free Experience | Pro Tease |
|-------|-----------------|-----------|
| Task completion | "Great job!" | "Pro users complete 3x faster" |
| Limit reached | "You're at your limit" | "Pro unlocks unlimited" |
| Feature discovery | "Preview this feature" | "Upgrade to access fully" |

### Communication Style by Audience

| Audience | Tone | Focus | Avoid |
|----------|------|-------|-------|
| End Users | Friendly, helpful | Benefits, ease | Technical jargon |
| Admins | Professional, precise | Control, security | Marketing fluff |
| Executives | Confident, strategic | ROI, outcomes | Implementation details |
| Developers | Technical, direct | Capabilities, APIs | Vague claims |

## Decision Framework

| Situation | Recommendation | Rationale |
|-----------|---------------|-----------|
| New feature announcement | Lead with user benefit, not feature specs | Users care about outcomes, not capabilities |
| Upgrade prompt needed | Show value gap, not just limitation | Positive framing converts better than restriction |
| Enterprise sales collateral | Focus on transformation and partnership | Executives buy strategic outcomes |
| User facing limit | Acknowledge constraint, offer path forward | Frustration converted to opportunity |
| Success story needed | Find tenant with quantified outcomes | Numbers make stories credible and shareable |
| Downgrade scenario | Preserve relationship for future return | Today's downgrade is tomorrow's upgrade prospect |
