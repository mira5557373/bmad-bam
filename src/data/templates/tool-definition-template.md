## Error Codes

| Code | HTTP Equivalent | Description | Resolution |
| ---- | --------------- | ----------- | ---------- |
| INVALID_INPUT | 400 | Input validation failed | Check parameter constraints |
| NOT_FOUND | 404 | Resource not found | Verify resource exists |
| PERMISSION_DENIED | 403 | Insufficient permissions | Check tenant/user permissions |
| RATE_LIMITED | 429 | Too many requests | Wait and retry |
| INTERNAL_ERROR | 500 | Internal tool error | Contact support |

## Verification Checklist

### Tool Definition Completeness

- [ ] Tool name follows naming conventions (lowercase, hyphenated)
- [ ] Input schema includes all required parameters with types and constraints
- [ ] Output schema defines success and error response structures
- [ ] All parameter validation rules documented with error messages
- [ ] Rate limits defined for all subscription tiers (FREE, PRO, ENTERPRISE)

### Multi-Tenant Considerations

- [ ] Tenant context requirement explicitly documented (required/optional)
- [ ] Cross-tenant access policy defined and justified
- [ ] Tier-based access restrictions documented
- [ ] Tool automatically scopes queries by tenant_id from context

### Security and Observability

- [ ] Required permissions listed with descriptions
- [ ] Kill switch configuration documented for emergency disable
- [ ] Metrics defined for monitoring (calls, duration, errors)
- [ ] Audit logging captures tenant_id and sanitized parameters
- [ ] Input sanitization and output filtering approaches specified

- [ ] Usage examples provided for common scenarios
- [ ] Integration guidelines include agent and cross-tool patterns
- [ ] Test coverage requirements specified (unit, integration, isolation)
- [ ] Dependencies documented (internal modules, external services)
- [ ] Deprecation policy and version history maintained
