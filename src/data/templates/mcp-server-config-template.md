## Server Architecture

### Deployment Model

| Model | Description | Use Case |
|-------|-------------|----------|
| Shared Server | Single MCP server with tenant filtering | Cost-efficient, standard isolation |
| Per-Tenant Server | Dedicated MCP server per tenant | Enterprise, maximum isolation |
| Hybrid | Shared Free/Pro, dedicated Enterprise | Balanced approach |

### Server Configuration

```yaml
mcp_server:
  name: "{{project_name}}-mcp"
  version: "1.0.0"
  transport: "stdio"  # or "http", "websocket"
  
  tenant_isolation:
    enabled: true
    strategy: "context_injection"  # or "server_per_tenant"
    context_header: "X-Tenant-ID"
  
  resource_limits:
    max_concurrent_requests: 100
    request_timeout_ms: 30000
    max_response_size_kb: 1024
```

## Tool Registry

### Available Tools

| Tool Name | Description | Scope | Tier Availability |
|-----------|-------------|-------|-------------------|
| database_query | Execute tenant-scoped queries | Tenant | All |
| file_access | Access tenant file storage | Tenant | Pro+ |
| api_call | Make external API calls | Tenant | Enterprise |
| admin_tools | Platform administration | Platform | Admin only |

### Tool Permissions

```yaml
tool_permissions:
  database_query:
    allowed_tiers: ["free", "pro", "enterprise"]
    tenant_scoped: true
    requires_approval: false
    rate_limit: 100/minute
    
  api_call:
    allowed_tiers: ["enterprise"]
    tenant_scoped: true
    requires_approval: true
    allowed_domains:
      - "*.tenant-domain.com"
      - "api.approved-service.com"
```
