### Role Configuration

```yaml
roles:
  platform_admin:
    scope: global
    permissions:
      - "*:*:*"  # All permissions
    constraints: []
    
  tenant_admin:
    scope: tenant
    permissions:
      - "tenant:*:*"
      - "users:*:*"
      - "resources:*:*"
    constraints:
      - tenant_id: "{{current_tenant}}"
    
  user:
    scope: tenant
    permissions:
      - "resources:read:*"
      - "resources:create:own"
      - "resources:update:own"
      - "resources:delete:own"
    constraints:
      - tenant_id: "{{current_tenant}}"
      - owner_id: "{{current_user}}"
```

## Permissions

### Permission Format

```
<resource>:<action>:<scope>

Examples:
- users:read:all         - Read all users
- users:read:own         - Read own user
- resources:create:any   - Create any resource
- resources:delete:own   - Delete own resources
```

## Attribute-Based Conditions

### Supported Attributes

| Attribute          | Type                   | Source                 | Usage                  |
| ------------------ | ---------------------- | ---------------------- | ---------------------- |
| tenant_id          | UUID                   | JWT/Header             | Tenant isolation       |
| user_id            | UUID                   | JWT                    | Ownership              |
| role               | String                 | JWT                    | Role-based             |
| tier               | Enum                   | Tenant config          | Feature access         |
| department         | String                 | User profile           | Data access            |
| time               | DateTime               | System                 | Time-based access      |

### Policy Examples

```yaml
policies:
  - name: "tenant_isolation"
    effect: deny
    condition:
      attribute: tenant_id
      operator: not_equals
      value: "{{resource.tenant_id}}"
    
  - name: "own_resource"
    effect: allow
    resources: ["resources:*:own"]
    condition:
      attribute: user_id
      operator: equals
      value: "{{resource.owner_id}}"
    
  - name: "business_hours"
    effect: deny
    resources: ["sensitive:*:*"]
    condition:
      attribute: time
      operator: not_in_range
      value: ["09:00", "17:00"]
```
