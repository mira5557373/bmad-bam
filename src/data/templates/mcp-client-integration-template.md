## Client Architecture

### Connection Patterns

| Pattern | Description | Use Case |
|---------|-------------|----------|
| Direct Client | One client per agent | Simple scenarios |
| Connection Pool | Shared pool with tenant routing | High throughput |
| Gateway Client | Central gateway with fan-out | Enterprise scale |

### Client Configuration

```yaml
mcp_client:
  server_url: "mcp://server.{{project_name}}.internal"
  transport: "http"
  
  connection_pool:
    min_connections: 5
    max_connections: 50
    idle_timeout_ms: 60000
    
  tenant_routing:
    header_name: "X-Tenant-ID"
    context_propagation: true
```

## Error Handling

### Retry Strategy

```yaml
error_handling:
  retry:
    max_attempts: 3
    backoff: "exponential"
    base_delay_ms: 100
    max_delay_ms: 5000
    
  circuit_breaker:
    failure_threshold: 5
    reset_timeout_ms: 30000
    
  fallback:
    enabled: true
    strategy: "graceful_degradation"
```

### Error Categories

| Error Type | Retry | User Message | Logging |
|------------|-------|--------------|---------|
| Connection failure | Yes | "Service temporarily unavailable" | Error |
| Permission denied | No | "Action not permitted" | Warning |
| Rate limited | Yes (with backoff) | "Please try again later" | Info |
| Timeout | Yes | "Request took too long" | Warning |
