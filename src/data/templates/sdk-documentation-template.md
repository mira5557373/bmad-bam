### OAuth 2.0 Authentication

#### Authorization Code Flow

```typescript
// Step 1: Redirect to authorization
const authUrl = client.auth.getAuthorizationUrl({
  clientId: 'your-client-id',
  redirectUri: 'https://your-app.com/callback',
  scope: ['read:resources', 'write:resources'],

const tokens = await client.auth.exchangeCode({
  code: 'authorization-code',
  redirectUri: 'https://your-app.com/callback',
});

// Step 3: Use access token
const authenticatedClient = new {{sdk_class}}({
  accessToken: tokens.accessToken,
  tenantId: 'tenant-uuid',
});
```

#### Token Refresh

```typescript
const newTokens = await client.auth.refreshToken({
  refreshToken: tokens.refreshToken,
});
```

## Tenant Context Setup

### Tenant Context Overview

Every SDK operation requires tenant context to ensure proper data isolation. The tenant context is established during client initialization and propagated with every request.

### Initialization Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `apiKey` | string | Yes* | API key for authentication |
| `accessToken` | string | Yes* | OAuth access token (alternative to API key) |
| `tenantId` | string | Yes | Target tenant identifier |
| `environment` | string | No | `production` or `sandbox` (default: `production`) |
| `baseUrl` | string | No | Custom API base URL |
| `timeout` | number | No | Request timeout in milliseconds |
| `retries` | number | No | Number of retry attempts |

*Either `apiKey` or `accessToken` is required.

### Environment Configuration

| Environment | Base URL | Purpose | Data Isolation |
|-------------|----------|---------|----------------|
| Production | `{{production_url}}` | Live operations | Production data |
| Sandbox | `{{sandbox_url}}` | Testing and development | Isolated test data |
| Local | `{{local_url}}` | Local development | Local database |

```typescript
// Production environment
const prodClient = new {{sdk_class}}({
  apiKey: process.env.PROD_API_KEY,
  tenantId: 'tenant-uuid',
  environment: 'production',
});

// Sandbox environment (for testing)
const sandboxClient = new {{sdk_class}}({
  apiKey: process.env.SANDBOX_API_KEY,
  tenantId: 'tenant-uuid',
  environment: 'sandbox',
});
```

### Tenant Context Propagation

The SDK automatically propagates tenant context in all requests:

| Layer | Propagation Method | Verification |
|-------|-------------------|--------------|
| HTTP Headers | `X-Tenant-ID` | Server-side validation |
| Request Body | `tenantId` field (where applicable) | Schema validation |
| Query Parameters | `tenant_id` (where applicable) | Parameter validation |

```typescript
// Tenant context is automatically included
const resources = await client.resources.list(); // Includes tenant context

// Verify current tenant context
const currentTenant = await client.tenants.getCurrent();
console.log(`Operating as tenant: ${currentTenant.id}`);
```

## Code Examples

### Basic CRUD Operations

#### Create Resource

```typescript
const newResource = await client.resources.create({
  name: 'My Resource',
  description: 'Resource description',
  metadata: {
    customField: 'value',
  },
});

console.log(`Created resource: ${newResource.id}`);
```

#### Read Resource

```typescript
// Get single resource
const resource = await client.resources.get('resource-uuid');

// List resources with pagination
const resources = await client.resources.list({
  page: 1,
  pageSize: 20,
  sort: 'createdAt:desc',
});

console.log(`Found ${resources.total} resources`);
```

#### Update Resource

```typescript
const updatedResource = await client.resources.update('resource-uuid', {
  name: 'Updated Name',
  description: 'Updated description',
});
```

#### Delete Resource

```typescript
await client.resources.delete('resource-uuid');
```

### Pagination

```typescript
// Using page-based pagination
const page1 = await client.resources.list({ page: 1, pageSize: 50 });
const page2 = await client.resources.list({ page: 2, pageSize: 50 });

// Using cursor-based pagination
let cursor: string | undefined;
const allResources: Resource[] = [];

do {
  const response = await client.resources.list({
    cursor,
    limit: 100,
  });
  
  allResources.push(...response.data);
  cursor = response.nextCursor;
} while (cursor);

// Using async iterator (recommended)
for await (const resource of client.resources.listAll()) {
  console.log(resource.name);
}
```

### Filtering and Searching

```typescript
// Filter resources
const filteredResources = await client.resources.list({
  filter: {
    status: 'active',
    createdAt: { gte: '2026-01-01' },
    tags: { contains: 'important' },
  },
});

// Search resources
const searchResults = await client.resources.search({
  query: 'search term',
  fields: ['name', 'description'],
});
```

### AI Agent Execution

```typescript
// Execute an AI agent
const execution = await client.agents.execute({

}
```

## Error Handling

### Error Types

| Error Class | HTTP Status | Description | Recovery |
|-------------|-------------|-------------|----------|
| `AuthenticationError` | 401 | Invalid or expired credentials | Refresh token or re-authenticate |
| `AuthorizationError` | 403 | Insufficient permissions | Check API key scopes |
| `NotFoundError` | 404 | Resource not found | Verify resource ID |
| `ValidationError` | 400 | Invalid request data | Check request parameters |
| `RateLimitError` | 429 | Rate limit exceeded | Wait and retry |
| `TenantError` | 400/403 | Tenant context issue | Verify tenant ID |
| `ServerError` | 5xx | Server-side error | Retry with backoff |

### Error Handling Example

```typescript
import {
  {{sdk_class}},
  AuthenticationError,
  RateLimitError,
  ValidationError,
  TenantError,
} from '@{{org}}/{{sdk_package}}';

try {
  const resource = await client.resources.get('resource-uuid');
} catch (error) {
  if (error instanceof AuthenticationError) {
    console.error('Authentication failed. Please check your API key.');
    // Refresh credentials
  } else if (error instanceof RateLimitError) {
    console.error(`Rate limited. Retry after ${error.retryAfter} seconds.`);
    // Wait and retry
  } else if (error instanceof ValidationError) {
    console.error('Validation errors:', error.errors);
    // Fix request data
  } else if (error instanceof TenantError) {
    console.error('Tenant context error:', error.message);
    // Verify tenant ID
  } else {
    console.error('Unexpected error:', error);
    throw error;
  }
}
```

## Rate Limits

### Rate Limit Tiers

| Tier | Requests/Minute | Requests/Day | Concurrent | Burst |
|------|----------------|--------------|------------|-------|
| Free | 60 | 1,000 | 5 | 10 |
| Pro | 300 | 50,000 | 20 | 50 |
| Enterprise | 1,000 | Unlimited | 100 | 200 |

### Rate Limit Headers

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed |
| `X-RateLimit-Remaining` | Requests remaining in window |
| `X-RateLimit-Reset` | Unix timestamp when limit resets |
| `Retry-After` | Seconds to wait before retrying (on 429) |

### Rate Limit Handling

```typescript
// Check rate limit status
const rateLimitInfo = client.getRateLimitInfo();
console.log(`Remaining: ${rateLimitInfo.remaining}/${rateLimitInfo.limit}`);

// Handle rate limiting automatically
const client = new {{sdk_class}}({
  apiKey: 'your-api-key',
  tenantId: 'tenant-uuid',
  rateLimiting: {
    enabled: true,
    queueRequests: true, // Queue requests instead of failing
    maxQueueSize: 100,
  },
});
```

## Multi-Tenant Considerations

### Tenant ID Propagation

The SDK ensures tenant ID is propagated in all requests:

```typescript
// Tenant context from initialization
const client = new {{sdk_class}}({
  apiKey: 'api-key',
  tenantId: 'primary-tenant-uuid',
});

// All operations use the initialized tenant context
await client.resources.list(); // Uses 'primary-tenant-uuid'

// Temporary tenant context switch (Enterprise only)
const results = await client.withTenant('other-tenant-uuid').resources.list();
```

### Tier-Based Features

```typescript
// Check available features for current tenant tier
const features = await client.features.list();

if (features.includes('ai-agents')) {
  // AI agent functionality available
  await client.agents.execute({ agentId: 'agent-uuid', input: {} });
} else {
  console.log('AI agents not available on current tier');
}

// Feature flags per tier
const featureFlags = await client.features.getFlags();
console.log('SSO enabled:', featureFlags.sso);
console.log('Custom branding:', featureFlags.customBranding);
```

### Sandbox vs Production

| Aspect | Sandbox | Production |
|--------|---------|------------|
| Data | Test data, can be reset | Real customer data |
| Rate Limits | Relaxed | Tier-based |
| Webhooks | Test endpoints | Production endpoints |
| Billing | No charges | Metered billing |
| AI Agents | Mock responses available | Full execution |

```typescript
// Environment-aware configuration
const client = new {{sdk_class}}({
  apiKey: process.env.API_KEY,
  tenantId: process.env.TENANT_ID,
  environment: process.env.NODE_ENV === 'production' ? 'production' : 'sandbox',
});

// Check current environment
const env = client.getEnvironment();
console.log(`Operating in ${env} environment`);
```

### Cross-Tenant Operations (Enterprise)

```typescript
// Parent tenant with cross-tenant access
const parentClient = new {{sdk_class}}({
  apiKey: 'parent-api-key',
  tenantId: 'parent-tenant-uuid',
  allowCrossTenant: true,

// List child tenants
```
