## Test Case Structure

### Test Case Template

| Field              | Required               | Description            |
| ------------------ | ---------------------- | ---------------------- |
| Test ID            | Yes                    | Unique identifier      |
| Title              | Yes                    | Descriptive name       |
| Category           | Yes                    | Unit/Integration/E2E   |
| Priority           | Yes                    | P1-P4                  |
| Preconditions      | Yes                    | Required state         |
| Steps              | Yes                    | Execution steps        |
| Expected Result    | Yes                    | Success criteria       |
| Actual Result      | On execution           | Observed outcome       |
| Status             | On execution           | Pass/Fail/Blocked      |

### Example Test Case

```yaml
test_case:
  id: TC-{{module}}-{{number}}
  title: "{{test_title}}"
  category: {{test_category}}
  priority: {{test_priority}}
  module: {{module_name}}
  
  preconditions:
    - {{precondition_1}}
    - {{precondition_2}}
  
  steps:
    - step: 1
      action: "{{action_1}}"
      expected: "{{expected_1}}"
    - step: 2
      action: "{{action_2}}"
      expected: "{{expected_2}}"
  
  expected_result: "{{overall_expected}}"
  
  tags:
    - {{tag_1}}
    - {{tag_2}}
```
