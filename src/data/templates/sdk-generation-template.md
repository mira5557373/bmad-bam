## Error Handling

### Error Types

| Error Class        | HTTP Codes             | Retryable          | Description        |
| ------------------ | ---------------------- | ------------------ | ------------------ |
| AuthenticationError| 401                    | No                 | Invalid credentials|
| AuthorizationError | 403                    | No                 | Insufficient perms |
| NotFoundError      | 404                    | No                 | Resource not found |
| ValidationError    | 400, 422               | No                 | Invalid input      |
| RateLimitError     | 429                    | Yes                | Rate limited       |
| ServerError        | 500-599                | Yes                | Server issues      |
| TenantError        | 403                    | No                 | Tenant context     |

### Error Response Structure

```typescript
interface SDKError {
  code: string;           // Machine-readable error code
  message: string;        // Human-readable message
  statusCode: number;     // HTTP status code
  tenantId?: string;      // Affected tenant
  requestId: string;      // For support tracking
  retryAfter?: number;    // Seconds until retry (429)
  details?: object;       // Additional context
}
```
