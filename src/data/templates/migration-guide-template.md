## Pre-Migration Checklist

### Environment Readiness

- [ ] Source environment documented and accessible
- [ ] Target environment provisioned and validated
- [ ] Network connectivity verified between environments
- [ ] Required credentials and access permissions secured
- [ ] Monitoring and alerting configured

### Data Readiness

- [ ] Source data inventory completed
- [ ] Data quality assessment passed
- [ ] Data mapping documentation complete
- [ ] Sensitive data handling procedures defined
- [ ] Backup strategy tested and verified

### Team Readiness

- [ ] Migration team roles assigned
- [ ] Escalation paths documented
- [ ] Communication channels established
- [ ] On-call schedule confirmed
- [ ] Stakeholder notification completed

- [ ] Dependencies identified and sequenced
- [ ] Migration scripts tested in staging environment
- [ ] Rollback procedures documented and tested
- [ ] Performance baselines established

### Rollback Decision Criteria

| Trigger | Threshold | Decision |
|---------|-----------|----------|
| Error rate | > {{error_threshold}}% | Initiate rollback |
| Data integrity failure | Any | Immediate rollback |
| Performance degradation | > {{perf_threshold}}% | Assess and decide |
| Tenant isolation breach | Any | Immediate rollback |
| Critical functionality failure | Any | Initiate rollback |

### Point of No Return

**After Step {{ponr_step}}, rollback requires full re-migration.**

Ensure explicit approval before proceeding past this point.
