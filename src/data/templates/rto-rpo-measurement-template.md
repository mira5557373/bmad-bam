## RTO Measurement Methodology

### Definition and Scope

**Recovery Time Objective (RTO):** The maximum acceptable time that a service can be offline after a disaster before business operations are critically impacted.

| Scope Element          | Definition                             |
| ---------------------- | -------------------------------------- |
| Start Point            | {{rto_start_def}}                      |
| End Point              | {{rto_end_def}}                        |
| Measurement Trigger    | {{rto_trigger}}                        |
| Success Criteria       | {{rto_success}}                        |

### RTO Measurement Components

| Component              | Description                            | Measurement Method     |
| ---------------------- | -------------------------------------- | ---------------------- |
| Detection Time         | Time to detect the failure             | {{det_method}}         |
| Decision Time          | Time to decide on DR activation        | {{dec_method}}         |
| Failover Time          | Time to execute failover procedures    | {{fo_method}}          |
| Recovery Time          | Time to restore services               | {{rec_method}}         |
| Validation Time        | Time to verify service restoration     | {{val_method}}         |

### RTO Calculation Formula

```
Total RTO = Detection Time + Decision Time + Failover Time + Recovery Time + Validation Time
```

| Component              | Description                            | Measurement Method     |
| ---------------------- | -------------------------------------- | ---------------------- |
