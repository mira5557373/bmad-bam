## API Key Architecture Overview

### 1.1 Key Types

| Key Type | Purpose | Lifetime | Scope | Use Case |
|----------|---------|----------|-------|----------|
| Service Key | Server-to-server auth | Long-lived | Service-specific | Backend integrations |
| User Key | User-associated access | Medium-lived | User permissions | Developer access |
| Temporary Key | Short-term access | Minutes to hours | Limited | CI/CD, testing |
| Tenant Admin Key | Tenant management | Long-lived | Tenant-wide | Admin operations |
| Read-Only Key | Data retrieval only | Configurable | Read scopes | Reporting, analytics |

### 1.2 Key Structure

```
┌─────────────────────────────────────────────────────────────────┐
│                    API Key Structure                             │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                    Key Format                              │   │
│  │  prefix_version_keyid_secret                               │   │
│  │    │       │      │      │                                 │   │
│  │    │       │      │      └─ Random secret (32 bytes)       │   │
│  │    │       │      └──────── Key identifier (UUID)          │   │
│  │    │       └─────────────── Version indicator (v1, v2)     │   │
│  │    └─────────────────────── Environment prefix             │   │
│  │                             (pk = prod, sk = staging)       │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  Example: pk_v1_abc123_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx          │
│                                                                  │
│  Storage: Only hash stored, full key shown once at creation     │
└─────────────────────────────────────────────────────────────────┘
```

## Scope Definitions

### 3.1 Scope Hierarchy

```
┌─────────────────────────────────────────────────────────────────┐
│                    Scope Hierarchy                               │
│                                                                  │
│  admin:*                                                         │
│    ├── admin:read              (Read admin settings)             │
│    ├── admin:write             (Modify admin settings)           │
│    └── admin:delete            (Delete admin resources)          │
│                                                                  │
│  tenant:*                                                        │
│    ├── tenant:read             (Read tenant data)                │
│    ├── tenant:write            (Create/update tenant data)       │
│    └── tenant:users            (Manage tenant users)             │
│                                                                  │
│  agents:*                                                        │
│    ├── agents:read             (View agent configs)              │
│    ├── agents:execute          (Run agents)                      │
│    ├── agents:write            (Create/modify agents)            │
│    └── agents:memory           (Access agent memory)             │
│                                                                  │
│  data:*                                                          │
│    ├── data:read               (Read business data)              │
│    ├── data:write              (Create/update data)              │
│    └── data:delete             (Delete data)                     │
│                                                                  │
│  billing:*                                                       │
│    ├── billing:read            (View billing info)               │
│    └── billing:manage          (Modify billing)                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Scope Definitions

| Scope | Description | Risk Level | Requires Approval |
|-------|-------------|------------|-------------------|
| admin:* | Full admin access | Critical | Yes |
| admin:read | Read admin settings | Medium | No |
| admin:write | Modify admin settings | High | Yes |
| tenant:read | Read tenant data | Low | No |
| tenant:write | Modify tenant data | Medium | No |
| agents:execute | Execute AI agents | Medium | {{agent_approval}} |
| agents:memory | Access agent memory | High | Yes |
| data:read | Read business data | Low | No |
| data:write | Create/update data | Medium | No |
| data:delete | Delete data | High | {{delete_approval}} |
| billing:manage | Manage billing | High | Yes |

### 3.3 Scope by Key Type

| Key Type | Default Scopes | Maximum Scopes | Restricted Scopes |
|----------|----------------|----------------|-------------------|
| Service Key | {{service_default_scopes}} | {{service_max_scopes}} | None |
| User Key | {{user_default_scopes}} | {{user_max_scopes}} | admin:*, billing:manage |
| Temporary Key | {{temp_default_scopes}} | {{temp_max_scopes}} | admin:*, agents:memory |
| Admin Key | {{admin_default_scopes}} | All | None |
| Read-Only Key | data:read, tenant:read | Read scopes only | All write/delete |

### 3.4 Scope Validation Rules

| Rule | Description | Enforcement |
|------|-------------|-------------|
| Scope subset | Key scopes must be subset of user permissions | Creation time |
| Wildcard expansion | * scopes expand to all sub-scopes | Runtime |
| Tenant boundary | Scopes apply within tenant only | Runtime |
| Cross-tenant | Requires explicit cross-tenant scope | Creation + runtime |
| Hierarchy | Child scopes inherit parent restrictions | Runtime |

```
┌─────────────────────────────────────────────────────────────────┐
│                    Key Rotation Flow                             │
│                                                                  │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐        │
│  │  Current    │     │  Generate   │     │  Grace      │        │

│  └─────────────┘     └─────────────┘     └──────┬──────┘        │
│                                                  │               │
│                                           ┌──────▼──────┐        │
│                                           │  Deprecate  │        │
│                                           │  Old Key    │        │
│                                           │  (v1)       │        │

│                                                  │               │
│                                           ┌──────▼──────┐        │
│                                           │  Revoke     │        │
│                                           │  Old Key    │        │
│                                           │  (v1)       │        │

│                                                                  │
│  Timeline:                                                       │
│  ───────────────────────────────────────────────────────────►   │
│  │ v2 created │ Grace (both) │ v1 deprecated │ v1 revoked │     │
│  │            │  {{grace}}   │ {{deprecate}} │             │     │
└─────────────────────────────────────────────────────────────────┘
```

### 9.1 Key Management API

|----------|--------|-------------|------|
| /api/v1/keys | POST | Create new key | admin:write or tenant:write |
