## Definitions

### 1.1 Core Definitions

| Term | Definition |
|------|------------|
| Personal Data | Any information relating to an identified or identifiable natural person |
| Data Subject | An identified or identifiable natural person whose personal data is processed |
| Processing | Any operation performed on personal data (collection, storage, use, disclosure, erasure) |
| Data Controller | The entity determining the purposes and means of processing personal data |
| Data Processor | The entity processing personal data on behalf of the Controller |
| Sub-processor | A third party engaged by the Processor to process personal data |
| Data Breach | A breach of security leading to accidental or unlawful destruction, loss, alteration, or unauthorized disclosure of personal data |
| Tenant | {{tenant_name}} - a customer of the multi-tenant SaaS platform |

### 1.2 Multi-Tenant Specific Definitions

| Term | Definition |
|------|------------|
| Tenant Isolation | Technical and organizational measures ensuring data separation between tenants |
| Tenant Context | Runtime identification and scoping mechanism for tenant-specific data access |
| Shared Infrastructure | Platform components used by multiple tenants with logical separation |
| Tenant-Specific Processing | Data processing activities unique to a specific tenant's requirements |

## Roles and Responsibilities

### 2.1 Data Controller Responsibilities (Tenant)

The Data Controller (Tenant) agrees to:

| Responsibility | Description | Compliance Reference |
|---------------|-------------|---------------------|
| Lawful Basis | Ensure valid legal basis exists for all processing activities | GDPR Art. 6 |
| Data Subject Rights | Facilitate exercise of data subject rights | GDPR Art. 12-22 |
| Accuracy | Ensure personal data is accurate and kept up to date | GDPR Art. 5(1)(d) |
| Purpose Limitation | Process data only for specified, explicit purposes | GDPR Art. 5(1)(b) |
| Instructions | Provide documented processing instructions to Processor | GDPR Art. 28(3)(a) |
| Notification | Notify Processor of any changes to processing requirements | GDPR Art. 28 |

### 2.2 Data Processor Responsibilities (Platform)

The Data Processor (Platform) agrees to:

| Responsibility | Description | Compliance Reference |
|---------------|-------------|---------------------|
| Instruction Compliance | Process data only on documented instructions from Controller | GDPR Art. 28(3)(a) |
| Confidentiality | Ensure authorized personnel are bound by confidentiality | GDPR Art. 28(3)(b) |
| Security Measures | Implement appropriate technical and organizational measures | GDPR Art. 32 |
| Sub-processor Management | Engage sub-processors only with prior authorization | GDPR Art. 28(2) |
| Assistance | Assist Controller with data subject requests and compliance | GDPR Art. 28(3)(e-f) |
| Audit Support | Make available information demonstrating compliance | GDPR Art. 28(3)(h) |
| Data Return/Deletion | Return or delete data at end of services | GDPR Art. 28(3)(g) |

- [ ] Submit to audits and inspections

## Data Breach Notification

### 8.1 Breach Classification

| Severity | Definition | Examples | Notification Timeline |
|----------|------------|----------|----------------------|
| Critical | High risk to rights and freedoms | Mass data exposure, financial data breach | Immediate (< 4 hours) |
| High | Likely risk to rights and freedoms | PII unauthorized access | 24 hours |
| Medium | Possible risk | Encrypted data exposure | 48 hours |
| Low | Unlikely risk | Failed breach attempt | 72 hours |

### 8.2 Notification Process

| Step | Responsible Party | Timeline | Deliverable |
|------|------------------|----------|-------------|
| Detection | Platform | T+0 | Incident ticket |
| Assessment | Platform | T+4h | Severity classification |
| Controller Notice | Platform | Per severity | Breach notification form |
| Authority Notice | Tenant | 72 hours | Supervisory authority report |
| Subject Notice | Tenant | Without undue delay | Data subject communication |
| Remediation | Joint | Ongoing | Incident report |

### 8.3 Breach Notification Content

The Platform shall provide the following information:

- [ ] Nature of the breach (categories and approximate number of data subjects/records)
- [ ] Contact details of Data Protection Officer
- [ ] Likely consequences of the breach
- [ ] Measures taken or proposed to address the breach
- [ ] Measures to mitigate possible adverse effects
- [ ] Tenant-specific impact assessment
