## Metric Calculations

### 3.1 Availability Calculation

```
┌─────────────────────────────────────────────────────────────────┐
│                 Availability Calculation                         │
│                                                                  │
│  Monthly Uptime % = (Total Minutes - Downtime Minutes) × 100    │
│                     ────────────────────────────────────         │
│                            Total Minutes                         │
│                                                                  │
│  Where:                                                          │
│    Total Minutes = Days in month × 24 × 60                       │
│    Downtime = Unplanned outages + Maintenance overruns           │
│                                                                  │
│  Exclusions (not counted as downtime):                           │
│    - Scheduled maintenance within window                         │
│    - Third-party service outages                                 │
│    - Customer-caused issues                                      │
│    - Force majeure events                                        │
│                                                                  │
│  Example (30-day month):                                         │
│    Total = 43,200 minutes                                        │
│    Downtime = 22 minutes                                         │
│    Uptime = (43,200 - 22) / 43,200 × 100 = 99.949%              │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Response Time Calculation

| Metric | Formula | Notes |
|--------|---------|-------|
| P50 (Median) | 50th percentile of response times | Half faster, half slower |
| P95 | 95th percentile | 95% of requests faster |
| P99 | 99th percentile | SLA compliance metric |
| Average | Sum(times) / Count | Can be skewed by outliers |

**Measurement Points:**
- Start: Request received at API gateway
- End: Response fully transmitted
- Excludes: Client network latency

### 6.2 Breach Response Process

```
┌─────────────────────────────────────────────────────────────────┐
│                 Breach Response Flow                             │
│                                                                  │
│  1. Detection                                                    │
│     └── Automated alert triggers                                 │
│                                                                  │
│  2. Triage (< {{triage_sla}})                                   │
│     └── Assess impact, identify affected tenants                 │
│                                                                  │
│  3. Communication                                                │
│     ├── Internal: {{internal_notify}}                           │
│     └── External: {{external_notify}} (if threshold met)        │
│                                                                  │
│  4. Resolution                                                   │
│     └── Technical remediation                                    │
│                                                                  │
│  5. Verification                                                 │
│     └── Confirm SLA restored                                     │
│                                                                  │
│  6. Documentation                                                │
│     └── RCA within {{rca_sla}}                                  │
│                                                                  │
│  7. Credit Processing                                            │
│     └── Calculate and apply within {{credit_sla}}               │
└─────────────────────────────────────────────────────────────────┘
```
