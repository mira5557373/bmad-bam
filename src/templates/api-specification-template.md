## Authentication

### Supported Authentication Methods

| Method       | Type         | Header/Parameter        | Description                     |
| ------------ | ------------ | ----------------------- | ------------------------------- |
| API Key      | apiKey       | X-API-Key               | Static API key authentication   |
| OAuth 2.0    | oauth2       | Authorization: Bearer   | OAuth 2.0 with JWT tokens       |
| JWT          | http         | Authorization: Bearer   | Direct JWT authentication       |

### Security Schemes

```yaml
components:
  securitySchemes:
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key
      description: API key for service-to-service authentication
    
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
      description: JWT token for user authentication
    
    OAuth2:
      type: oauth2
      flows:
        authorizationCode:
          authorizationUrl: {{auth_url}}/authorize
          tokenUrl: {{auth_url}}/token
          refreshUrl: {{auth_url}}/refresh
          scopes:
            read: Read access to resources
            write: Write access to resources
            admin: Administrative access
```

### Authentication Flow

1. Client requests access token from `{{auth_url}}/token`
2. Token includes tenant_id claim for multi-tenant context
3. All API requests include `Authorization: Bearer {token}`
4. Token validated and tenant context extracted per request

## Error Codes

### Standard Error Response

```yaml
components:
  schemas:
    ErrorResponse:
      type: object
      required:
        - error
      properties:
        error:
          type: object
          required:
            - code
            - message
          properties:
            code:
              type: string
              description: Machine-readable error code
            message:
              type: string
              description: Human-readable error message
            details:
              type: array
              items:
                type: object
                properties:
                  field:
                    type: string
                  reason:
                    type: string
            request_id:
              type: string
              description: Request tracking identifier
            timestamp:
              type: string
              format: date-time
```

### Error Code Reference

| HTTP Status | Error Code              | Description                          | Resolution                           |
| ----------- | ----------------------- | ------------------------------------ | ------------------------------------ |
| 400         | INVALID_REQUEST         | Malformed request body               | Check request format                 |
| 400         | VALIDATION_ERROR        | Field validation failed              | Review field constraints             |
| 400         | INVALID_PARAMETER       | Invalid query/path parameter         | Check parameter values               |
| 401         | UNAUTHORIZED            | Missing or invalid authentication    | Provide valid credentials            |
| 401         | TOKEN_EXPIRED           | Authentication token expired         | Refresh token                        |
| 403         | FORBIDDEN               | Insufficient permissions             | Request elevated access              |
| 403         | TENANT_MISMATCH         | Resource belongs to another tenant   | Verify resource ownership            |
| 404         | NOT_FOUND               | Resource does not exist              | Check resource identifier            |
| 409         | CONFLICT                | Resource state conflict              | Resolve conflict and retry           |
| 422         | UNPROCESSABLE_ENTITY    | Semantic validation failed           | Review business rules                |
| 429         | RATE_LIMITED            | Too many requests                    | Wait and retry with backoff          |
| 500         | INTERNAL_ERROR          | Unexpected server error              | Contact support                      |
| 502         | BAD_GATEWAY             | Upstream service error               | Retry later                          |
| 503         | SERVICE_UNAVAILABLE     | Service temporarily unavailable      | Retry with exponential backoff       |
| 504         | GATEWAY_TIMEOUT         | Upstream service timeout             | Retry later                          |

## Rate Limiting

### Tier-Based Rate Limits

| Tier       | Requests/Minute | Requests/Hour | Burst Limit | Concurrent |
| ---------- | --------------- | ------------- | ----------- | ---------- |
| FREE       | 60              | 1,000         | 10          | 5          |
| PRO        | 300             | 10,000        | 50          | 25         |
| ENTERPRISE | 1,000           | 100,000       | 200         | 100        |

### Rate Limit Headers

| Header                    | Description                              |
| ------------------------- | ---------------------------------------- |
| X-RateLimit-Limit         | Maximum requests allowed in window       |
| X-RateLimit-Remaining     | Requests remaining in current window     |
| X-RateLimit-Reset         | Unix timestamp when window resets        |
| X-RateLimit-Retry-After   | Seconds to wait before retrying (429)    |

### Rate Limit Response


## Multi-Tenant Headers

### Required Headers

| Header        | Required | Description                              | Example                    |
| ------------- | -------- | ---------------------------------------- | -------------------------- |
| X-Tenant-ID   | Yes*     | Tenant identifier (if not in JWT)        | `tenant_abc123`            |
| Authorization | Yes      | Bearer token with tenant context         | `Bearer eyJhbGci...`       |
| X-Request-ID  | No       | Client-provided request tracking ID      | `req_client_xyz`           |
| X-Idempotency-Key | No   | Idempotency key for POST/PUT requests    | `idem_abc123`              |

*Note: X-Tenant-ID is extracted from JWT `tenant_id` claim if not provided.

### Response Headers

| Header           | Description                               |
| ---------------- | ----------------------------------------- |
| X-Request-ID     | Server-generated request tracking ID      |
| X-Tenant-ID      | Confirmed tenant context                  |
| X-Response-Time  | Request processing time in milliseconds   |

### Tenant Context Propagation

```yaml
components:
  parameters:
    TenantHeader:
      name: X-Tenant-ID
      in: header
      required: false
      schema:
        type: string
        format: uuid
      description: |
        Tenant identifier. Optional if using JWT authentication
        with tenant_id claim. When provided, must match JWT claim.
```

### Tenant Isolation Guarantees

| Aspect                | Enforcement                              |
| --------------------- | ---------------------------------------- |
| Data Access           | RLS policy filters by tenant_id          |
| Cross-Tenant Requests | Rejected with 403 TENANT_MISMATCH        |
| Audit Logging         | All requests logged with tenant context  |
| Resource Quotas       | Enforced per-tenant based on tier        |

## Usage Examples

### Example 1: Create Resource

**Request:**
```bash
curl -X POST "{{base_url}}/{{endpoint_path}}" \
  -H "Authorization: Bearer {{token}}" \
  -H "Content-Type: application/json" \
  -H "X-Idempotency-Key: idem_unique_123" \
  -d '{
    "{{field_name}}": "{{field_value}}"
  }'
```

**Response:**
```json
{
  "id": "{{resource_id}}",
  "tenant_id": "{{tenant_id}}",
  "{{field_name}}": "{{field_value}}",
  "created_at": "{date}-04-07T10:30:00Z",
  "updated_at": "{date}-04-07T10:30:00Z"
}
```

### Example 2: List Resources with Pagination

**Request:**
```bash
curl -X GET "{{base_url}}/{{endpoint_path}}?page=1&limit=20&sort=created_at&order=desc" \
  -H "Authorization: Bearer {{token}}"
```

**Response:**
```json
{
  "data": [
    {
      "id": "{{resource_id}}",
      "{{field_name}}": "{{field_value}}"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "total_pages": 8,
    "has_next": true,
    "has_previous": false
  }
}
```
