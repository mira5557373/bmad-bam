## TenantContext Structure

### Core Fields

| Field | Type | Required | Source |
|-------|------|----------|--------|
| `tenant_id` | UUID | Yes | JWT claim `tenant_id` |
| `user_id` | UUID | No | JWT claim `sub` |
| `workspace_id` | UUID | No | JWT claim `workspace_id` |
| `tier` | Enum | Yes | JWT claim `tier` |
| `permissions` | Array | No | JWT claim `permissions` |
| `correlation_id` | UUID | Yes | Header or generated |
| `ip_address` | String | No | Request origin |

### Context Factory

```
TenantContext.from_jwt(token: str) -> TenantContext
TenantContext.from_job(payload: dict) -> TenantContext
TenantContext.system() -> SystemContext
```

## HTTP Request Propagation

### Entry Point

| Step | Action | Implementation |
|------|--------|----------------|
| 1 | Extract JWT | `Authorization: Bearer {token}` |
| 2 | Validate signature | Auth service |
| 3 | Build TenantContext | Context factory |
| 4 | Attach to request | `request.state.tenant_context` |
| 5 | Set RLS variable | `SET LOCAL app.current_tenant` |

### Middleware Configuration

```
Middleware Order:
1. Correlation ID (generate if missing)
2. Authentication (JWT validation)
3. Tenant Context (build from JWT)
4. RLS Setup (set database variable)
5. Application Logic
```

## Database Propagation

### RLS Setup

| Database | Method | Statement |
|----------|--------|-----------|
| PostgreSQL | Session variable | `SET LOCAL app.current_tenant = :tenant_id` |
| PostgreSQL | Per-query | `WHERE tenant_id = :tenant_id` |

### Connection Pool Requirements

- [ ] RLS variable reset on connection return
- [ ] No connection sharing between tenants
- [ ] Transaction-scoped tenant setting

| `Authorization` | Service token | Service authentication |

### gRPC Calls

| Metadata Key | Value |
|--------------|-------|
| `tenant-id` | TenantContext.tenant_id |
| `correlation-id` | TenantContext.correlation_id |

## Validation Points

### Mandatory Checks

| Location | Validation | On Failure |
|----------|------------|------------|
| Request entry | JWT present | 401 Unauthorized |
| Context build | tenant_id non-null | 400 Bad Request |
| Database query | RLS active | Abort query |
| Event publish | tenant_id in headers | Reject event |
| Job execution | Context valid | Fail job |

## Error Handling

| Error | Code | HTTP Status | Handling |
|-------|------|-------------|----------|
| `MissingTenantContext` | `TENANT_001` | 401 | Reject request |
| `InvalidTenantContext` | `TENANT_002` | 400 | Reject request |
| `TenantMismatch` | `TENANT_003` | 403 | Log and reject |
| `ContextDeserializationError` | `TENANT_004` | 500 | Fail and alert |
