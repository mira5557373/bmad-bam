## Versioning Strategy

### Semantic Versioning Rules

| Version Component | Increment When | Example |
|------------------|----------------|---------|
| MAJOR (X.0.0) | Breaking changes that require client updates | 2.0.0 → 3.0.0 |
| MINOR (0.X.0) | New features, backward-compatible additions | 2.1.0 → 2.2.0 |
| PATCH (0.0.X) | Bug fixes, security patches, documentation | 2.1.1 → 2.1.2 |

### API Version Selection

| Method | Format | Example |
|--------|--------|---------|
| URL Path | `/api/v{major}/` | `/api/v3/tenants` |
| Header | `X-API-Version: {version}` | `X-API-Version: 3.2.1` |
| Query Parameter | `?api_version={version}` | `?api_version=3.2.1` |

### Tenant Version Assignment

| Tenant Tier | Default Version | Version Lock | Auto-Upgrade |
|-------------|-----------------|--------------|--------------|
| Free | Latest stable | No | Yes (forced) |
| Pro | Latest stable | Yes (1 version) | Opt-in |
| Enterprise | Tenant choice | Yes (any supported) | Manual |
