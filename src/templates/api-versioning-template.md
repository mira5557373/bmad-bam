## Breaking Change Policy

### What Constitutes Breaking Changes

| Change Type        | Breaking               | Migration Path         |
| ------------------ | ---------------------- | ---------------------- |
| Remove endpoint    | YES                    | Version bump required  |
| Remove field       | YES                    | Deprecation period     |
| Change field type  | YES                    | New field + deprecate  |
| Add required field | YES                    | Default value          |
| Add optional field | NO                     | Additive change        |
| Add new endpoint   | NO                     | Additive change        |

## Multi-Tenant Considerations

### Tenant Version Preferences

| Tier               | Version Selection      | Auto-Upgrade           |
| ------------------ | ---------------------- | ---------------------- |
| Free               | Latest only            | Automatic              |
| Pro                | Current or Previous    | Opt-in                 |
| Enterprise         | Any supported          | Manual                 |

### Version Override Rules

| Scenario           | Behavior               | Override Allowed       |
| ------------------ | ---------------------- | ---------------------- |
| New tenant         | Latest stable          | No                     |
| Security patch     | Force upgrade          | No                     |
| Minor version      | Preserve preference    | Yes                    |
| Major version      | Require migration      | Yes (with notice)      |
