# V2 Gap Fulfillment Design Spec

**Date:** 2026-04-28
**Status:** Approved
**Approach:** Hybrid Migration + Generation

---

## Problem Statement

BAM V2 has structural gaps that prevent full workflow execution:
- 40 template stubs with `[Content]` placeholders (56 lines each)
- 3 missing CSV registry files
- ~15 missing quality gate checklists (29 exist, 44 defined)
- 6 minimal patterns needing expansion

## Solution: Hybrid Migration + Generation

### Template Tier Strategy

| Tier | Count | Templates | Strategy |
|------|-------|-----------|----------|
| **Core** | 10 | master-architecture, module-architecture, tenant-isolation, agent-runtime, facade-contract, convergence-report, production-readiness, requirements, triage-report, module-epics | Migrate from V1 + enhance |
| **Operational** | 15 | billing-design, compliance-mapping, caching-strategy, data-residency, event-architecture, incident-response, observability, scaling, security-architecture, tenant-onboarding, tenant-offboarding, white-labeling, cost-model, capacity-plan, decision-log | Generate from patterns + domains |
| **Workflow** | 15 | agent-debug-report, agent-trace, api-version, cross-module-story, gate-checklist, integration-test-plan, llm-version, memory-tier, mcp-tool-contract, research-report, tool-contract, epic-story, story-tasks, validation-report, migration-plan | Synthesize from workflow step files |

### CSV Registry Files

| File | Source | Content |
|------|--------|---------|
| `bam-patterns.csv` | Migrate from V1 `src/data/bam-patterns.csv` | Pattern decision criteria + web queries |
| `compliance-frameworks.csv` | Migrate from V1 `src/data/compliance-frameworks.csv` | GDPR, SOC2, HIPAA mapping |
| `section-pattern-map.csv` | Migrate from V1 `src/data/section-pattern-map.csv` | Template section → pattern mapping |

### Checklist Generation

Generate missing checklists from `quality-gates.csv`:
- Extract `gate_id`, `verification_tests`, `fail_recovery` columns
- Generate checklist structure matching existing `qg-f1.md` format
- Mark first 2-3 verification tests as CRITICAL

### Pattern Expansion

6 patterns need full BAM structure:
- When to Use / When NOT to Use
- Architecture diagram
- Trade-offs table
- Web Research Queries section

## Execution Order

1. **Phase 1: CSV Migration** - Copy 3 CSV files from V1
2. **Phase 2: Checklist Generation** - Generate ~15 missing checklists
3. **Phase 3: Core Templates** - Migrate 10 core templates from V1
4. **Phase 4: Operational Templates** - Generate 15 from patterns
5. **Phase 5: Workflow Templates** - Synthesize 15 from step files
6. **Phase 6: Pattern Expansion** - Expand 6 minimal patterns
7. **Phase 7: Test Updates** - Update assertion counts
